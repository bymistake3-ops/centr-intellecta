# Как проверить, что всё работает

## Smoke-тест: вся цепочка за 6 минут

Вместо того чтобы ждать сутки, можно временно переключить бота в режим, где
T1..T6 приходят каждую минуту.

### Подготовка

1. Заранее договорись с помощником (или используй второй аккаунт Telegram).
2. **Важно**: основной аккаунт админа и тестовый — разные. Не проверяй на
   аккаунте, который уже регистрировался.

### Тест

1. В Amvera → ENV → `SMOKE_TEST` = `true` → сохранить → **«Перезапустить»**.
2. Подожди 20 сек, пока бот поднимется.
3. С тестового аккаунта в Telegram: открой бота → `/start` → «Поделиться номером».
4. Засеки 6 минут. Должны прийти 7 сообщений:
   - **T0 мгновенно:** welcome + PDF-бонус
   - **+1 минута:** T1 («завтра в 19:00 МСК»)
   - **+2 минуты:** T2 («сегодня в 19:00 МСК»)
   - **+3 минуты:** T3 («через час, вот ссылка»)
   - **+4 минуты:** T4 («стартуем через 5 минут»)
   - **+5 минут:** T5 («эфир идёт, ещё можно зайти»)
   - **+6 минут:** T6 («запись по ссылке»)
5. Проверь, что в каждом сообщении правильно подставлены имя, дата, ссылки.

### Что проверить ещё

**Защита от повторного /start.** С того же тестового аккаунта нажми `/start`
ещё раз — должен прийти текст «ты уже записан», без новых 7 сообщений.

**Кодовое слово.** Напиши боту кодовое слово (как оно указано в ENV, любым
регистром) — должна прийти ссылка на бесплатный курс.

**Неизвестный текст.** Напиши любую случайную фразу — бот должен ответить
подсказкой «нажми /start».

**Админ-статистика.** Со СВОЕГО аккаунта (`ADMIN_USER_ID`) напиши `/stats` —
придут цифры регистраций. С чужого аккаунта `/stats` ничего не ответит.

### Проверка переживания рестарта

Это самая важная проверка: что напоминания не теряются при перезапуске.

1. Остаёмся в режиме `SMOKE_TEST=true`.
2. С тестового аккаунта: `/start` → поделиться контактом.
3. Как только получил welcome (T0), **сразу нажми «Перезапустить»** в Amvera.
4. В течение 6 минут всё равно должны прийти все T1..T6.

Если не пришли — включено ли `DATA_DIR=/data` в ENV? Включён ли persistent disk?

### После теста

1. Amvera → ENV → `SMOKE_TEST` = `false` → сохранить → **«Перезапустить»**.
2. Тестовый аккаунт: можешь попросить его заблокировать бота, а потом
   разблокировать и снова `/start`, чтобы удалить тестовую регистрацию из БД.
   Либо удалить запись из БД вручную (попроси разработчика).

## Проверка production-режима (один цикл)

После smoke-теста полезно один раз пройти всю цепочку в реальном времени:

1. `SMOKE_TEST=false` (должно быть после smoke-теста).
2. В 11:00 МСК зарегистрируйся с тестового аккаунта.
3. Ожидания:
   - сразу: welcome + PDF
   - в 20:00 сегодня: T1 (напоминание «завтра в 19:00»)
   - завтра в 10:00: T2
   - завтра в 18:00: T3
   - завтра в 18:55: T4
   - завтра в 19:30: T5
   - послезавтра в 19:00: T6

Edge-case: если регистрация **после 20:00** — T1 автоматически пропускается
(время уже прошло). T2..T6 всё равно придут.

## Что смотреть в логах

Amvera → проект → вкладка **«Логи»**.

- `Bot started as @...` — бот поднялся.
- `messages.yaml validated` — тексты валидны.
- `Scheduler started` — планировщик запущен.
- `scheduled N touches for user X` — регистрация прошла, напоминания запланированы.
- `sent TX to Y` — напоминание отправлено.
- `user Y blocked bot, marking and cancelling remaining jobs` — юзер
  заблокировал бота, оставшиеся напоминания отменены.

Если в логах `ValidationError` на старте — сломан `messages.yaml`. Если
`Invalid token` — проблема с `BOT_TOKEN`.
