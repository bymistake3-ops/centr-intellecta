# Как проверить, что всё работает

## Smoke-тест: вся цепочка за 4 минуты

Вместо того чтобы ждать сутки, можно временно переключить бота в режим, где
TG2..TG5 приходят каждую минуту.

### Подготовка

- Для теста используй тестовый Telegram-аккаунт, отличный от аккаунта админа.
- Не гоняй тест на аккаунте, который уже `/start`-ал — бот ответит «вы уже в списке».

### Прогон

1. Amvera → ENV → `SMOKE_TEST` = `true` → сохрани → **«Перезапустить»**.
2. Подожди 20 сек, пока бот поднимется.
3. С тестового аккаунта в Telegram: `@intellectcentr_masterclass_bot` → `/start`.
4. Должны прийти **мгновенно**:
   - TG1: приветствие с датой/временем вебинара
   - TG1a: `bonus.pdf` + подпись «Карта из 5 шагов»
5. Далее каждую минуту:
   - **+1 мин:** TG2 («сегодня в 19:00 мастер-класс»)
   - **+2 мин:** TG3 («до старта 30 минут» + кнопка «Открыть эфир»)
   - **+3 мин:** TG4 («начинаем через 5 минут» + кнопка «Открыть эфир»)
   - **+4 мин:** TG5 («эфир закончился, отправьте ПРОКАЧКА за бонусами»)

### Что проверить дополнительно

**Защита от повторного `/start`.** С того же тестового аккаунта `/start` ещё раз —
придёт текст «вы уже в списке», без новых TG1/TG1a.

**Кодовое слово ПРОКАЧКА.** В любой момент отправь боту `ПРОКАЧКА` (любой регистр,
или `prokachka`) — должны прийти:
- сообщение TG6 с кнопкой «Зарегистрироваться на курс»
- `checklist.pdf` с подписью «Чек-лист 30 дней прокачки мозга»

После активации ПРОКАЧКА оставшиеся TG-джобы этого юзера **отменяются** — он
больше не получит напоминаний. Проверить: отправь ПРОКАЧКА сразу после `/start`,
подожди 5 минут — TG2..TG5 прийти не должны.

**Случайный текст.** Напиши любую фразу, не похожую на ПРОКАЧКА — бот ответит:
«Чтобы получить бонусы, отправьте слово ПРОКАЧКА».

**Админ-статистика.** Со СВОЕГО аккаунта (`ADMIN_USER_ID`) — `/stats` → цифры
регистраций. С чужого аккаунта `/stats` молчит.

### Переживание рестарта

Важная проверка: напоминания не теряются при перезапуске.

1. Режим `SMOKE_TEST=true`.
2. Тестовый `/start` → получил TG1 + TG1a.
3. Сразу в Amvera нажми **«Перезапустить»**.
4. Остальные TG2..TG5 всё равно должны прийти в течение 4 минут.

Если не пришли — проверь `DATA_DIR=/data` в ENV и включён ли persistent disk.

### Обработка блокировки бота

1. С тестового аккаунта: `/start`.
2. Сразу заблокируй бота в Telegram.
3. В логах Amvera: `user X blocked bot, marking and cancelling remaining jobs`.
4. В БД (через `/stats`) — количество заблокировавших увеличится.

### После теста

1. Amvera → ENV → `SMOKE_TEST` = `false` → Restart.
2. Опционально: попроси разработчика очистить тестовую запись из БД.

## Production-проверка (один реальный цикл)

После smoke-теста полезно пройти цикл в реальном времени:

1. `SMOKE_TEST=false`.
2. В 11:00 МСК зарегистрируйся с тестового аккаунта.
3. Ожидания:
   - сразу: TG1 + TG1a
   - завтра в 10:00 МСК: TG2
   - завтра в 18:30 МСК: TG3 (−30 мин до эфира)
   - завтра в 18:55 МСК: TG4 (−5 мин)
   - завтра в 20:30 МСК: TG5 (+90 мин после старта; вебинар длится 75 мин)

## Что смотреть в логах

Amvera → проект → **«Логи»**.

- `Bot started as @intellectcentr_masterclass_bot` — бот поднят.
- `messages.yaml validated` — тексты валидны.
- `Scheduler started (smoke_test=...)` — планировщик жив.
- `registered user X, webinar at ... UTC` — регистрация прошла.
- `scheduled N touches for user X` — напоминания запланированы.
- `sent TGx to Y` — напоминание отправлено.
- `user Y blocked bot, marking and cancelling remaining jobs` — обработан 403.
- `user X activated secret word — remaining jobs cancelled` — сработал ПРОКАЧКА.

Частые ошибки:
- `ValidationError` на старте — сломан `messages.yaml`. Откати в истории GitHub.
- `Invalid token` — проблема с `BOT_TOKEN`. Обнови в ENV.
