# Деплой на Amvera Cloud — пошагово

Эта инструкция — для владельца без технического опыта. Каждый шаг подробно.

Amvera выбран потому что: российские карты и СБП, деплой из GitHub без SSH,
встроенный persistent disk для базы данных, Python 3.11 из коробки, управление
переменными окружения из браузера.

## Шаг 1. Получить токен бота

1. Открой в Telegram бот **@BotFather**.
2. Отправь `/newbot`.
3. Придумай имя (как будет отображаться) и username (должен заканчиваться на `bot`).
4. BotFather пришлёт сообщение с токеном вида `123456789:AAH...`.
5. **Скопируй токен и сохрани в заметки.** Никому не показывай — кто знает токен, управляет ботом.

> Если когда-нибудь токен утечёт — @BotFather → `/revoke` → получишь новый.

## Шаг 2. Узнать свой Telegram ID

1. Открой бот **@userinfobot** и нажми `/start`.
2. В ответе будет строка **Id: 123456789** — это твой числовой Telegram ID.
3. Сохрани в заметки. Понадобится для команды `/stats`.

## Шаг 3. Зарегистрироваться на Amvera

1. Открой https://amvera.ru
2. Нажми «Регистрация» → войди по email или Telegram.
3. Подтверди email (если просил).

## Шаг 4. Создать проект

1. В панели Amvera нажми **«Создать проект»** → **«Приложение»**.
2. Окружение: **Python**, версия **3.11**.
3. Способ: **«Подключить GitHub»**. Авторизуй Amvera в GitHub (появится окно).
4. Выбери репозиторий **`bymistake3-ops/centr-intellecta`**.
5. Ветка: **`main`** (или `claude/add-webinar-email-reminders-k3QXY`, если ещё не слили).
6. Стартовая команда (если поле есть): `python -m app.main` — обычно уже взята из `amvera.yml`.

## Шаг 5. Включить persistent disk

1. В проекте → вкладка **«Хранилище»** (или «Storage»).
2. Включи встроенный диск.
3. Точка монтирования: **`/data`**
4. Размер: **1 ГБ** (с запасом)

Это важно: база данных (регистрации пользователей и расписание напоминаний)
хранится на этом диске. Без него каждый рестарт всё теряется.

## Шаг 6. Переменные окружения

Открой вкладку **«Переменные окружения»** и добавь по одной каждую из списка.
Значения возьми из своих заметок или спроси разработчика.

| Переменная | Что указать |
|---|---|
| `BOT_TOKEN` | Токен от BotFather (шаг 1) |
| `ADMIN_USER_ID` | Твой Telegram ID (шаг 2) |
| `WEBINAR_URL` | Ссылка на комнату автовебинара |
| `RECORD_URL` | Ссылка на запись. Пока пусто — поставь `https://example.com/record`, обновишь позже |
| `FREE_COURSE_URL` | Ссылка на бесплатный курс по кодовому слову |
| `SECRET_WORD` | Твоё кодовое слово (например `ИНТЕЛЛЕКТ`) |
| `DATA_DIR` | `/data` |
| `TZ` | `Europe/Moscow` |
| `LOG_LEVEL` | `INFO` |
| `SMOKE_TEST` | `false` |

## Шаг 7. Первый деплой

1. Нажми **«Собрать и запустить»**.
2. В логе сборки должны появиться строки:
   - `pip install -r requirements.txt`
   - `Successfully installed aiogram ... APScheduler ... SQLAlchemy ...`
   - `Bot started as @твой_бот`
3. Если видишь ошибку — скопируй последние 20 строк и покажи разработчику.

## Шаг 8. Проверить в Telegram

1. Открой своего бота в Telegram (по username, который придумал в шаге 1).
2. Нажми `/start`.
3. Должен появиться текст приветствия и кнопка **«Поделиться номером»**.
4. Поделись контактом.
5. Должно прийти: текст welcome + файл `bonus.pdf` (если ты его загрузил в репо).

Если PDF не приходит — положи файл `content/bonus.pdf` в репозиторий через
GitHub Web UI (см. `EDITING.md`).

## Шаг 9. Первый эфир — что делать

Перед первым эфиром:
- проверь, что `WEBINAR_URL` указывает на рабочую комнату

После эфира:
- возьми ссылку на запись
- в Amvera → ENV → `RECORD_URL` → вставь ссылку → **«Перезапустить»**
- все T6 (через 24 часа после вебинара) подхватят новую ссылку автоматически

## Частые проблемы

**«Bot started» в логе, но /start не отвечает.** Проверь, что в BotFather этот
токен НЕ включён в другого клиента (старая копия бота на другом хостинге). Один
токен — один процесс.

**«Invalid token»** — опечатка в `BOT_TOKEN`, или токен был revoke-нут. Получи
новый у BotFather и обнови ENV.

**`ValidationError: messages.yaml`** — в `content/messages.yaml` пропал ключ
или сломался отступ. Откати последний коммит в GitHub Web UI (история файла →
кнопка «Revert»).

**Бот не шлёт напоминания.** Проверь, что persistent disk (`/data`) включён.
Если его нет — после рестарта теряется база и все расписанные джобы.
