# Деплой на Amvera Cloud — пошагово

Эта инструкция — для владельца без технического опыта. Каждый шаг подробно.

Amvera выбран потому что: российские карты и СБП, деплой из GitHub без SSH,
встроенный persistent disk для базы данных, Python 3.11 из коробки, управление
переменными окружения из браузера.

## Шаг 1. Токен бота

1. Открой в Telegram бот **@BotFather**.
2. Если бот уже создан (`@intellectcentr_masterclass_bot`) — отправь
   `/token` → выбери этого бота → получи токен. Если где-то токен утёк
   (например, был в переписке), сначала сделай `/revoke` — получишь новый.
3. Если бота ещё нет — `/newbot`, следуй подсказкам.
4. Токен вида `123456789:AAH...` сохрани в заметки. Никому не показывай.

## Шаг 2. Свой Telegram ID

1. Открой бот **@userinfobot** и нажми `/start`.
2. В ответе будет строка **Id: 123456789** — это твой числовой ID.
3. Сохрани. Понадобится для команды `/stats`.

## Шаг 3. Регистрация на Amvera

1. Открой https://amvera.ru
2. «Регистрация» → вход по email или Telegram.
3. Подтверди email.

## Шаг 4. Создать проект

1. **«Создать проект»** → **«Приложение»**.
2. Окружение: **Python**, версия **3.11**.
3. **«Подключить GitHub»** → авторизуй Amvera.
4. Выбери репо **`bymistake3-ops/centr-intellecta`**.
5. Ветка: **`main`** (после того как этот PR вольётся в main).
6. Стартовая команда: обычно берётся из `amvera.yml`. Если поле пустое — `python -m app.main`.

## Шаг 5. Persistent disk

1. Вкладка **«Хранилище»**.
2. Включи встроенный диск.
3. Точка монтирования: **`/data`**
4. Размер: **1 ГБ**.

Без диска при каждом рестарте теряются регистрации и запланированные напоминания.

## Шаг 6. Переменные окружения

Вкладка **«Переменные окружения»** → добавь по одной:

| Переменная | Что указать |
|---|---|
| `BOT_TOKEN` | Токен от BotFather |
| `ADMIN_USER_ID` | Твой числовой Telegram ID |
| `WEBINAR_URL` | Ссылка на комнату автовебинара (трансляция «5 навыков мозга») |
| `COURSE_URL` | Ссылка на регистрацию на 6-дневный курс «Прокачка мозга» |
| `SECRET_WORD` | `ПРОКАЧКА` |
| `DATA_DIR` | `/data` |
| `TZ` | `Europe/Moscow` |
| `LOG_LEVEL` | `INFO` |
| `SMOKE_TEST` | `false` |

## Шаг 7. Загрузить PDF-бонусы

Положи в репозиторий два PDF-файла:

- `content/bonus.pdf` — бонус за регистрацию (выдаётся в TG1a сразу после `/start`)
- `content/bonus2.pdf` — бонус после вебинара (выдаётся в TG6 после кодового слова ПРОКАЧКА)

Как загрузить через GitHub Web UI — см. `docs/EDITING.md`.

Если какого-то PDF нет — бот всё равно работает: просто вместо файла отправляется только текст, в логи пишется warning.

## Шаг 8. Первый деплой

1. Нажми **«Собрать и запустить»**.
2. В логе сборки должны появиться:
   - `pip install ...`
   - `Successfully installed aiogram ... APScheduler ... SQLAlchemy ...`
   - `messages.yaml validated`
   - `DB initialised at /data/bot.db`
   - `Scheduler started`
   - `Bot started as @intellectcentr_masterclass_bot`

## Шаг 9. Проверка в Telegram

1. Открой `@intellectcentr_masterclass_bot`.
2. `/start` → должны прийти два сообщения:
   - TG1: приветствие с датой и временем вебинара
   - TG1a: `bonus.pdf` с подписью «Карта из 5 шагов»

Если PDF не пришёл — значит `content/bonus.pdf` не загружен. Добавь его и передеплой не нужен — PDF читается с диска при каждом `/start`.

3. Отправь боту слово `ПРОКАЧКА` (в любом регистре) — должны прийти:
   - сообщение TG6 с кнопкой «Зарегистрироваться на курс»
   - `checklist.pdf` с подписью «Чек-лист 30 дней прокачки мозга»

## Шаг 10. Smoke-тест цепочки

См. `docs/TESTING.md`. Включаешь `SMOKE_TEST=true`, получаешь все TG2..TG5 за 4 минуты.

## Частые проблемы

**«Invalid token»** — опечатка в `BOT_TOKEN` или revoked. Обнови ENV и перезапусти.

**`ValidationError` при старте** — сломан `content/messages.yaml`. Откати файл к предыдущей версии через историю GitHub (Web UI → файл → History → Restore this version).

**Бот не шлёт напоминания.** Проверь, что persistent disk подключён и `DATA_DIR=/data`. Без диска `bot.db` создаётся в `/app` и стирается при рестарте.

**PDF не приходит.** Положи `content/bonus.pdf` (или `content/checklist.pdf`) в репозиторий. Файл до 10 МБ. Имя точное, в нижнем регистре.
