# Как редактировать контент без знания git

| Что меняешь | Где | Как задеплоится |
|---|---|---|
| Тексты сообщений | `content/messages.yaml` в GitHub | После commit → автодеплой Amvera |
| PDF-бонус «Карта из 5 шагов» | `content/bonus.pdf` в GitHub | После commit → автодеплой |
| PDF-чеклист «30 дней» | `content/checklist.pdf` в GitHub | После commit → автодеплой |
| Ссылки (эфир, курс) | Amvera → ENV | После Restart |
| Кодовое слово | Amvera → ENV `SECRET_WORD` | После Restart |

## Менять тексты сообщений

1. Открой https://github.com/bymistake3-ops/centr-intellecta
2. `content` → `messages.yaml`.
3. Карандаш (Edit this file).
4. Меняй **только текст внутри кавычек или после `|`**. Ключи (слова слева
   от `:`) и отступы НЕ трогай.

Пример правильной правки:

```yaml
tg1:
  text: |
    Здравствуйте.            # ✅ можно править этот текст
    Вы в списке на мастер-класс...
```

Пример неправильной правки:

```yaml
TG1:                         # ❌ менять ключ tg1 нельзя
  text:    Здравствуйте      # ❌ сбит отступ после `text:` — YAML сломается
```

5. Внизу — **«Commit changes»** → короткое описание → **Commit**.
6. Через 30–60 секунд Amvera задеплоит. Проверь в Telegram.

## Плейсхолдеры в текстах

Эти штуки в фигурных скобках подставляются автоматически:

| Плейсхолдер | Во что превратится |
|---|---|
| `{first_name}` | Имя из Telegram-профиля пользователя |
| `{webinar_date}` | Дата вебинара: «25 апреля» |
| `{webinar_time}` | Время МСК: «19:00» |
| `{webinar_url}` | Ссылка на эфир (из ENV `WEBINAR_URL`) |
| `{course_url}` | Ссылка на курс (из ENV `COURSE_URL`) |

Фигурные скобки обязательны. `first_name` без скобок — просто слово.

## Загрузить или заменить PDF

### `content/bonus.pdf` — «Карта из 5 шагов»

Выдаётся сразу после `/start` в сообщении TG1a.

1. В репо открой папку `content/`.
2. Если файл уже есть — клик по нему → три точки → **Delete file** → Commit.
3. Вернись в `content/` → **Add file → Upload files**.
4. Перетащи PDF. **Имя должно быть ровно `bonus.pdf`** (строчными буквами).
5. Commit.

### `content/checklist.pdf` — «Чек-лист 30 дней прокачки мозга»

Выдаётся в TG6 после кодового слова ПРОКАЧКА.

Та же процедура, имя файла ровно `checklist.pdf`.

Ограничение: файл до 10 МБ. Если больше — ужми PDF (например https://smallpdf.com/compress-pdf).

## Менять ссылки

1. Amvera → свой проект → **«Переменные окружения»**.
2. Найди переменную и впиши новое значение:
   - `WEBINAR_URL` — ссылка на эфир
   - `COURSE_URL` — ссылка на регистрацию на курс «Прокачка мозга»
3. Сохрани → **«Перезапустить»** (Restart).
4. Через 10–30 сек бот подхватит.

## Менять кодовое слово

Amvera → ENV → `SECRET_WORD` → новое слово → Restart.

По ТЗ стоит `ПРОКАЧКА`. Если хочешь сменить — учти, что тексты TG5 и фраза
«Чтобы получить бонусы, отправьте слово ПРОКАЧКА» жёстко написаны в
`messages.yaml`. Если меняешь кодовое слово — синхронно правь упоминания в yaml.

Бот принимает слово в любом регистре (`ПРОКАЧКА`, `прокачка`, `Прокачка`) и в
транслитерации (`prokachka`, `PROKACHKA`, `Prokachka`).

## Если что-то сломалось после правки

1. В GitHub открой историю файла (History).
2. Найди коммит ДО твоей правки.
3. Справа — **«...» → Restore this revision** → Commit.
4. Amvera задеплоит обратно за минуту.

## Что категорически нельзя

- Удалять файлы/папки в `app/`.
- Удалять ключи в `messages.yaml` (только значения).
- Коммитить `.env` с реальным токеном (он должен жить только в Amvera ENV).
