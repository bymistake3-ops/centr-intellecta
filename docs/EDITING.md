# Как редактировать контент без знания git

Весь контент, который ты, вероятно, захочешь менять, разделён на три категории:

| Что меняешь | Где | Как задеплоится |
|---|---|---|
| Тексты сообщений | `content/messages.yaml` в GitHub | После commit — автодеплой Amvera |
| PDF-бонус | `content/bonus.pdf` в GitHub | После commit — автодеплой Amvera |
| Ссылки (эфир, запись, курс) | Переменные окружения в Amvera | После «Перезапустить» |

## Менять тексты сообщений

1. Открой https://github.com/bymistake3-ops/centr-intellecta
2. Перейди в папку `content` → файл `messages.yaml`.
3. Справа вверху нажми иконку **карандаша** (Edit this file).
4. В открывшемся редакторе меняй **только текст внутри кавычек или после
   вертикальной черты `|`**. Ключи (слова слева от двоеточия) и отступы НЕ трогай.

Пример правильной правки:

```yaml
welcome:
  text: |
    Привет, {first_name}!
    ← сюда пиши свой текст          # ✅ OK, правим текст
    ...
```

Пример неправильной правки:

```yaml
welcome:                            # ❌ нельзя менять ключ welcome
привет:                             # ❌ добавлять свои ключи тоже нельзя
  text:    Привет!                  # ❌ сбит отступ — YAML сломается
```

5. Прокрути вниз до блока **«Commit changes»**.
6. В поле описания напиши коротко что поменял (например: «T2 — новый текст»).
7. Выбери **«Commit directly to the main branch»** (или в ту же ветку, откуда правишь).
8. Нажми зелёную кнопку **Commit changes**.
9. Через 30–60 секунд Amvera увидит push и задеплоит новую версию. Проверь в Telegram.

## Плейсхолдеры в текстах

Эти штуки в фигурных скобках подставляются автоматически:

| Плейсхолдер | Во что превратится |
|---|---|
| `{first_name}` | Имя юзера из его Telegram-профиля |
| `{webinar_date}` | Дата вебинара, например «25 апреля» |
| `{webinar_time}` | Время МСК, например «19:00» |
| `{webinar_url}` | Ссылка на эфир (из ENV `WEBINAR_URL`) |
| `{record_url}` | Ссылка на запись (из ENV `RECORD_URL`) |
| `{free_course_url}` | Ссылка на курс (из ENV `FREE_COURSE_URL`) |

Фигурные скобки обязательны. `first_name` без скобок — просто слово.

## Заменить PDF-бонус

1. Открой папку `content` в GitHub.
2. Кликни на файл `bonus.pdf`.
3. Справа вверху — три точки → **Delete file** → внизу Commit.
4. Вернись в папку `content`, нажми **Add file → Upload files**.
5. Перетащи новый PDF с компьютера на эту страницу.
6. **Важно: файл должен называться `bonus.pdf`** (строчными буквами, без пробелов).
7. Commit.

Ограничение: файл до 10 МБ. Если больше — предварительно ужми PDF
(например через https://smallpdf.com/compress-pdf).

## Менять ссылки (вебинар / запись / курс)

Ссылки хранятся в переменных окружения Amvera, чтобы при их смене не нужно
было трогать код.

1. Открой панель Amvera → свой проект.
2. Вкладка **«Переменные окружения»**.
3. Найди нужную:
   - `WEBINAR_URL` — ссылка на комнату автовебинара
   - `RECORD_URL` — ссылка на запись (обновляется после каждого эфира)
   - `FREE_COURSE_URL` — ссылка на бесплатный курс (выдаётся по кодовому слову)
4. Нажми на её значение, впиши новое, **сохрани**.
5. Нажми **«Перезапустить»** (Restart).
6. Через 10–30 сек бот подхватит новое значение. Проверь.

## Менять кодовое слово

Так же как ссылки: Amvera → ENV → `SECRET_WORD` → новое значение → Restart.

Слово нечувствительно к регистру и обрезает пробелы: `ИНТЕЛЛЕКТ`, `интеллект`,
`Интеллект  ` — всё сработает.

## Что категорически нельзя делать

- Удалять папку `app/` или файлы внутри.
- Удалять ключи в `messages.yaml`.
- Коммитить файл `.env` с реальными значениями токена (он должен быть только в Amvera ENV).
- Делать force push (если случайно узнал эту команду — не делай).

Если что-то сломалось после твоей правки:
1. В GitHub открой историю изменений нужного файла.
2. Найди предыдущий коммит (до твоей правки).
3. Справа вверху — **«...» → Restore this revision** → Commit.
4. Amvera задеплоит обратно за минуту.
